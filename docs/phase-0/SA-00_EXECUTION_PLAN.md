## 7. Security & RLS Baseline

### Principles
1. **RLS is mandatory** for all application tables (BE-00+).
2. **org_id** (or equivalent) on every org-scoped table.
3. Policies use `auth.uid()` or `auth.jwt() ->> '...'` for user context.
4. Service role bypasses RLS for backend operations; use only server-side.

### SA-00 RLS Baseline Migration
- No application tables exist yet. The baseline migration:
  - Documents the RLS strategy as comments.
  - Optionally creates a helper or schema placeholder.
  - Establishes that `row_level_security` is the default posture.

### RLS Policy Template (for future migrations)
```sql
-- Example pattern for org-scoped tables (BE-00 will define org model)
-- ALTER TABLE <table> ENABLE ROW LEVEL SECURITY;
-- CREATE POLICY "org_isolation" ON <table>
--   USING (org_id = (SELECT org_id FROM organization_members WHERE user_id = auth.uid() LIMIT 1));
```

### Audit Fields
- All tables: `created_at`, `updated_at` (timestamptz).
- Where relevant: `created_by`, `updated_by` (uuid referencing auth.users).

## 11. Recommended Scalable Architecture (BE-00 Hardening — Append)

### Single Origin via Next.js Reverse Proxy (Default)

We adopt the **most scalable** option for FE ↔ BE integration:

- **Frontend (Next.js) is the single public origin.** Browsers talk only to the Next.js server.
- Next.js proxies API calls to FastAPI via **rewrites** (not CORS from browser to FastAPI directly):
  - In dev: `/api/backend/*` → `http://localhost:8000/*` (or configured backend URL).
  - In production: same-origin `/api/backend/*` is rewritten to the FastAPI service URL.
- **Benefits:**
  - Eliminates most CORS complexity (browser sees one origin).
  - Simplifies auth: token can be sent same-origin; no cross-origin cookie issues.
  - Matches production deployment patterns (one ingress, backend hidden).
  - Clean separation: UI service vs API service; backend remains a separate process.

**Requirements when FE exists:**
- Add rewrites in `apps/web/next.config.*` (e.g. `async rewrites() { return [{ source: '/api/backend/:path*', destination: 'http://localhost:8000/:path*' }]; }` for dev).
- Prefer same-origin path (e.g. `/api/backend`) so no `NEXT_PUBLIC_API_BASE` is required for relative calls.
- Document that FastAPI remains a separate service but is hidden behind Next.js.

**Credentials:** Default is Bearer token model with `allow_credentials=false`. If cookie-based auth is introduced later, CORS and proxy behavior can be revisited and documented.
